css的三大特性：层叠性、继承性、优先级 相同的选择器给设置了相同的样式，此时一个样式就会覆盖（层叠）另一个冲突的样式，层叠性主要解决样式冲突的问题。
层叠性的原则： 样式冲突，遵循就近原则（就是离标签代码最近的那个有效），不冲突的样式，不会层叠 （也就是同一个标签写了重复的选择器，和重复的样式，哪个离代码近哪个生效，重复的选择器里面有些没有重复的样式不会被层叠掉，会生效）

子标签会继承父标签的某些样式，继承的优点：简化css代码，降低样式的复杂性 （也就是每个子标签的相同的样式，写在父标签里，子就不用每一个里面都写了，比如字体、颜色等）
子元素可以继承父元素的样式：基本都是继承文字属性样式（text、font、line 和 color）
行高的继承比较特殊，有两种写法，如下：
font：字体大小/行高px（如果子元素没有设置行高，会直接继承这个行高，如果子元素设置了行高，则不会继承了）
font：字体大小/1.5（这数字可以按需求写，后面不跟单位）（不带单位表示：当前子元素文字大小的1.5倍。如果子元素没有设置文字大小，会直接继承父的文字大小，再乘以1.5倍，就是当前子元素的行高了。如果子元素设置了字体大小，就直接设置的字体大小的1.5倍就是当前子元素的行高。

当同一个元素标签指定多个选择器，就会产生优先级。选择器相同，执行层叠性。选择器不同，则根据选择器的权重执行。 行内样式 > id选择器 > 类选择器 > 标签选择器 > 通用选择器

通用选择器，权重值0
标签选择器，权重值1
类选择器/伪类选择器，权重值都是10
id选择器，权重值100
行内样式表，权重值1000
!important,权重值无穷大
继承的权重是0，（不管父的权重多大，子继承样式时权重都是0） 例如a标签，浏览器默认给a标签指定了一个带下划线的蓝色样式，所以设置父标签的颜色时，a标签不会变成父标签设置的那个颜色，因为继承的权重是0，比浏览器默认指定a标签样式的权重小 计算权重：例如：p #abc .cde {} 这个后代选择器的权重是 1+100+10 = 111
